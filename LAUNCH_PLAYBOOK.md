# Launch Playbook (Website + Payment + Instant Download)

Goal: user pays on your site, downloads the app immediately, pastes a license key, and gets auto-updates.

## 1) Host downloads (recommended)
Host these on a stable HTTPS URL (S3/R2/Cloudflare Pages):
- `appcast.xml`
- DMGs and any delta update files generated by Sparkle

Your app’s `SUFeedURL` should point to:
`https://YOUR-DOWNLOAD-DOMAIN/appcast.xml`

## 2) Generate Sparkle update keys (one-time, on your Mac)
1. Build once (this fetches Sparkle tools):
```bash
cd "/Users/jayantnahata/Desktop/ChatGPT Codex Folder/speakdash"
zsh scripts/build_app_bundle.sh
```
2. Generate/print your Sparkle public key:
```bash
zsh scripts/sparkle_generate_keys.sh
```
Copy the printed `SUPublicEDKey` into your release build env as `SPARKLE_PUBLIC_ED_KEY`.

## 3) Make a release “website drop” folder (DMG + appcast)
```bash

export SPARKLE_FEED_URL="https://YOUR-DOWNLOAD-DOMAIN/appcast.xml"
export SPARKLE_PUBLIC_ED_KEY="PASTE_FROM_generate_keys"
zsh scripts/release_prepare_website_drop.sh
```

Upload everything in:
`/Users/jayantnahata/Desktop/ChatGPT Codex Folder/speakdash/release/updates`
to your download host.

## 4) Payments + instant delivery
Fastest v1 approach:
- Use **Gumroad/LemonSqueezy** to take payment.
- Your checkout success page shows:
  - Download link to your DMG (hosted by you)
  - A license key (see below)

## 5) License keys (for paid access)
The app supports `licenseMode`:
- `off` (no license)
- `trial` (trial days, then requires license)
- `required` (always requires license)

You generate a license signing keypair once, keep private key secret, and embed the public key in config (`licensePublicKeyBase64`).
Use:
- `scripts/generate_license_key.swift` (issues a key from your private key)

In the app, users open `License...` and paste the key.

## 6) What to put on your website
- “Download for macOS” button (DMG link)
- “After install: open menu bar icon -> License -> paste key”
- FAQ: Permissions (Accessibility, Input Monitoring, Microphone)
